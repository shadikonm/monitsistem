# Руководство по запуску проекта (Currency Monitor)

Данный документ описывает пошаговый процесс развертывания и запуска проекта, включая базу данных, backend, frontend и AI-ассистента.

## 📋 Требования к окружению

Перед началом убедитесь, что у вас установлены:
- **Node.js** (рекомендуется версия 18.x или выше)
- **Docker** и **Docker Compose** (для запуска PostgreSQL)
- **Ollama** (опционально, для работы ИИ-ассистента)

---

## 🛠 Шаг 1: Подготовка конфигурации (Environment)

Вам необходимо создать файлы окружения (`.env`) для обеих частей проекта.

Откройте терминал в корневой папке проекта (`MPC`) и выполните:

```bash
# Копируем пример конфига для backend
cp backend/.env.example backend/.env

# Копируем пример конфига для frontend
cp frontend/.env.example frontend/.env
```

*(Вы можете открыть эти файлы `.env` и при необходимости изменить порты или ключи, но базовые значения обычно подходят для локального запуска).* 

---

## 🐘 Шаг 2: Запуск базы данных (PostgreSQL)

Проект использует PostgreSQL для хранения истории курсов. С помощью Docker запустить её можно одной командой.

В корневой папке выполните:

```bash
docker compose up -d
```

Эта команда скачает образ `postgres:16` (если его нет) и запустит контейнер `currency-postgres` в фоновом режиме на порту `5432`.

---

## ⚙️ Шаг 3: Запуск Backend (Серверная часть)

Серверная часть отвечает за сбор данных, их сохранение в БД и раздачу на frontend.

1. Перейдите в папку `backend`:
```bash
cd backend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите сервер в режиме разработки:
```bash
npm run dev
```

Если всё прошло успешно, вы увидите сообщение в терминале о том, что сервер запущен и подключен к базе данных.

---

## 🎨 Шаг 4: Запуск Frontend (Пользовательский интерфейс)

Для запуска клиентской части откройте **новый (второй) терминал**. Убедитесь, что вы находитесь в корне проекта (`MPC`).

1. Перейдите в папку `frontend`:
```bash
cd frontend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите приложение:
```bash
npm run dev
```

После этого Frontend будет доступен в браузере по адресу: [http://localhost:5173](http://localhost:5173) (порт может отличаться, смотрите вывод терминала).

---

## 🤖 Шаг 5: Настройка ИИ-ассистента (Опционально)

Если вы хотите полноценно использовать функции ИИ-окна в проекте, вам потребуется локально запущенный Ollama.

1. Скачайте и установите **Ollama** с официального сайта.
2. В терминале загрузите и запустите нужную модель (по умолчанию в backend указана `qwen3.5-cloud`):

```bash
ollama run qwen3.5-cloud
```

По умолчанию backend ожидает, что Ollama работает по адресу `http://localhost:11434`.

> **Примечание:** Если Ollama не запущен или недоступен, сервер перейдет в режим `fallback` и будет отвечать простыми заготовленными ответами.

---

## ✅ Резюме (для последующих запусков)

Когда проект уже настроен (файлы `.env` созданы, пакеты `npm install` установлены), повседневный запуск выглядит намного проще:

1. Терминал 1 (БД и Backend):
```bash
docker compose start
cd backend
npm run dev
```

2. Терминал 2 (Frontend):
```bash
cd frontend
npm run dev
```
